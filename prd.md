## 1. 서비스 개요

- **서비스 이름(가제)**: 라이브러리 썸네일 메이커
- **한 줄 소개**: 내 라이브러리(사진첩) 속 사진들을 직접 골라서, 3가지 레이아웃 중 하나로 정사각형 썸네일 이미지를 만들어 다운로드할 수 있는 서비스.
- **핵심 포인트**
  - “자동으로 골라주는 서비스”가 아니라,
  - **내가 좋아하는 사진을 직접 선택해서, 원하는 레이아웃으로 썸네일을 만드는 도구**라는 점이 중요하다.

---

## 2. 문제 정의

- 내 앨범 커버나 폴더 썸네일을 **내가 진짜 좋아하는 사진으로 예쁘게 꾸미고 싶다.**
- 하지만 보통 서비스들은 자동으로 한 장을 골라주거나, 마음에 안 드는 썸네일이 생기는 경우가 많다.
- 여러 장의 사진을 조합해서 **“나만의 썸네일 콜라주”**를 만들고 싶은데, 포토샵 같은 도구는 어렵고 귀찮다.
- 그래서,
  - **내 라이브러리에서 사진을 고르고,**
  - **준비된 3가지 레이아웃 중 하나를 선택하면,**
  - **정사각형 형태의 하나의 이미지로 합쳐서 다운로드할 수 있는 도구**가 필요하다.

---

## 3. 타겟 사용자

- **주요 사용자**
  - 내 리스트, 앨범, 폴더, 노션/블로그 표지 이미지 등을 예쁘게 꾸미고 싶은 사람
- **특징**
  - 디자인 툴(포토샵, 피그마 등)은 어렵거나 귀찮다.
  - “어려운 편집”보다는, **사진 선택 + 레이아웃 선택** 정도의 가벼운 편집을 선호한다.
  - 내 취향이 들어간 앨범 커버/표지 이미지를 만들고 싶다.

---

## 4. 주요 사용 시나리오

1. **앨범 커버 만들기**
   - 사용자는 “친구들과의 추억” 앨범을 새로 만들었다.
   - 앨범 커버를 멋지게 꾸미고 싶어서, 라이브러리에서 친구들과 찍은 사진 몇 장을 선택한다.
   - 3가지 레이아웃 중 하나를 고른 뒤, 정사각형 썸네일 이미지로 합쳐서 다운로드해서 앨범 커버로 설정한다.

2. **노션/블로그 표지 이미지 만들기**
   - 공부 기록이나 프로젝트 기록을 노션/블로그에 정리하고 있다.
   - 그 페이지의 표지 이미지를 예쁘게 하고 싶어서, 관련 사진들을 골라 썸네일을 만든다.
   - 완성된 정사각형 이미지를 다운로드해서 노션/블로그 표지로 사용한다.

3. **폴더/디렉토리 아이콘 이미지 만들기**
   - 컴퓨터 안에 ‘여행’, ‘학교’, ‘프로젝트’ 등 여러 폴더가 있다.
   - 각 폴더를 더 구분하기 쉽게, 폴더 아이콘처럼 쓸 수 있는 썸네일 이미지를 만든다.
   - 여행 사진 몇 장을 골라 썸네일을 만들고, 결과 이미지를 폴더 아이콘처럼 사용한다.

## 5. 핵심 기능

1. **3가지 레이아웃 제공**
   - **레이아웃 A**: 사진 1장 중심, 큰 정사각형(단일 이미지).
   - **레이아웃 B**: 사진 2~3장을 사용하는 간단한 분할 레이아웃 (예: 위/아래, 좌/우).
   - **레이아웃 C**: 사진 4장 이상을 사용하는 그리드형 콜라주 (예: 2x2, 3x3 등).
   - 각 레이아웃은 **결과물이 항상 정사각형**이 되도록 설계.

2. **내 라이브러리에서 사진 선택하기**
   - 사용자가 사진첩(또는 파일 업로드)을 통해 여러 장의 사진을 선택할 수 있다.
   - 앞에서 선택한 레이아웃에 필요한 사진 수만큼 선택해야 한다.
   - 선택된 사진들의 목록/미리보기 표시.

3. **레이아웃에 사진 배치하기**
   - 선택한 사진들을 레이아웃의 칸에 자동 배치.
   - 사용자는 필요하다면 사진의 위치를 바꾸거나, 어느 칸에 어떤 사진이 들어갈지 바꿀 수 있다(드래그 앤 드롭 등 단순 인터랙션).

4. **정사각형 썸네일 이미지 생성**
   - 레이아웃과 사진 배치를 기반으로 **1개의 정사각형 이미지**를 생성한다.
   - 모든 레이아웃의 최종 산출물은 정사각형 비율(1:1)을 유지한다.

5. **이미지 다운로드 기능**
   - 완성된 썸네일을 하나의 이미지 파일(JPEG/PNG 등)로 **다운로드**할 수 있다.
   - 파일 이름 예: `thumbnail-YYYYMMDD-HHMM.png`

6. **배경색/여백 설정**
   - 사용자가 썸네일의 배경색을 간단히 선택할 수 있다(예: 흰색, 검정, 파스텔 톤 몇 가지).
   - 사진 사이에 여백(간격)을 둘지 말지, 여백 두께를 어떻게 할지 간단하게 조정할 수 있다.

## 6. 화면(페이지) 구성 : 사용 흐름은 홈 → 레이아웃 선택 → 사진 선택 → 레이아웃 편집 → 결과/다운로드 순서로 진행한다.

1. **홈 화면**
   - 간단한 소개 문구:
     - “내 라이브러리 사진으로 정사각형 썸네일을 만들어보세요.”
   - “시작하기” 버튼
   - 간단한 예시 썸네일 이미지 2~3개 미리보기

2. **레이아웃 선택 화면**
   - 상단: 3가지 레이아웃 미리보기(작은 썸네일 형태).
   - 사용자가 레이아웃 A/B/C 중 하나를 선택.
- "다음으로" 버튼.


3. **사진 선택 화면**
   - 내 라이브러리(또는 파일 업로드 영역)를 보여준다.
   - 사진들을 그리드 형태(작은 정사각형 썸네일)로 나열한다.
   - 클릭으로 여러 장 선택 가능, 앞에서 선택한 레이아웃에 따라 몇 장 선택해야하는지 가이드 제공.
   - 라이브러리에서 선택한  개수 표시 (예: “5장 선택됨”).
   - “다음으로” 버튼.

4. **레이아웃 편집 화면**
   - 중앙: 선택한 레이아웃에 실제 사진들이 배치된 미리보기.
   - 사진 교체, 위치 바꾸기, 배경색/여백 설정.
   - “썸네일 생성하기” 버튼.

5. **결과/다운로드 화면**
   - 완성된 정사각형 썸네일 이미지 크게 보여주기.
   - “이미지 다운로드” 버튼.
   - “다시 만들기” 또는 “새 썸네일 만들기” 버튼.

---

## 7. 성공 기준 (지표)

- **기본 성공 기준**
  - 사용자가 **직접 사진을 선택하고, 레이아웃을 골라, 최종 이미지를 다운로드까지 하는 과정**을 끝까지 마치는 비율.
- **예시 수치**
  - 첫 주에 10명 이상의 사용자가 썸네일 이미지를 1개 이상 다운로드.
  - 썸네일 1개를 만드는 데 걸리는 평균 시간이 3분 이내.
  - 레이아웃 3가지 중, 적어도 2가지는 실제로 사용되는 비율이 30% 이상.

---

## 8. MVP 범위

- **레이아웃**
  - MVP에서는 레이아웃 A와 B만 제공한다.
  - 레이아웃 C(그리드형 콜라주)는 후속 버전에서 추가한다.

- **기능 범위**
  - 파일 업로드로 사진 선택.
  - 선택한 레이아웃에 사진 배치.
  - 정사각형 이미지 생성.
  - JPG 포맷으로 이미지 다운로드.

---

## 9. 기술/제한 사항 (간단 버전)

- **기술 스택**
  - 초기 버전은 **프론트엔드만**(HTML, CSS, JavaScript)으로 구현한다.
  - 브라우저에서 캔버스(Canvas)를 사용해 이미지를 합치고, 최종 이미지를 만든다.

- **이미지 입력 방식**
  - 초기 버전: 사용자가 직접 파일 업로드(PC/모바일에서 사진 선택). 휴대폰 사진첩에서 선택.

- **이미지 처리**
  - 브라우저에서 캔버스(Canvas)나 이미지 처리 라이브러리를 사용해,
    - 사진 크기 조정
    - 자르기(crop)
    - 레이아웃에 맞게 배치
    를 수행.
  - 최종 결과는 클라이언트(브라우저) 쪽에서 하나의 정사각형 이미지로 합쳐서 생성.

- **해상도**
  - 기본 해상도 예: 1080 x 1080 (정사각형, SNS/노션/블로그용으로 충분한 수준).

- **이미지 포맷**
  - 최종 산출물은 JPG 형식으로만 제공한다.

---

## 10. 인터랙션 규칙(버튼 활성화 조건)

- **“다음으로” 버튼 활성화 규칙**
  - 레이아웃 선택 화면:
    - 사용자가 레이아웃 A 또는 B 중 하나를 선택했을 때만 “다음으로” 버튼이 활성화된다.
  - 사진 선택 화면:
    - 사용자가 해당 레이아웃에 필요한 사진 수만큼(또는 최소 1장 이상) 선택했을 때만 “다음으로” 버튼이 활성화된다.
  - 그 외 화면에서도, 사용자가 필요한 필수 액션을 완료했을 때만 “다음으로” 또는 다음 단계로 가는 버튼이 활성화되도록 설계한다.
